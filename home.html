AB-Testing

For this project, I took two different versions of a cactus-shopping website, and performed A/B Testing and statistical analysis to gather insight into the difference between the two UI designs.


Part 0: Making the Websites

I was given a version of the cactus making website, and asked to make modifications to it to create a second version. I decided to make the shopping cart logo next to "CART" at the bottom of the screen the same color as the rest of the buttons on the screen, instead of the original red. Part 1: Creating Hypotheses

Users were assigned the task of adding $150 dollars worth of cacti into their cart. For this project, I measured two variables: Time to Completion and Return Rate. Time of Completion refers to the time it takes for the user to complete the task. The Return Rate refers to rate at which users decide to return to the original page. The following table shows my Null and Alternative Hypotheses for each task, as well as my reasoning for each.

Variable Null Hypothesis Alternative Hypothesis Reasoning
Time to Completion Version B and Version A have the same Time to Completion Version B has a longer Time to Completion than Version A Since Version B's cart icon has the same color as the rest of the buttons on the page, the user could confuse the two elements, and take longer to figure out where to view the contents of the cart.
Return Rate Version A and Version B have the same Return Rate Version B has a lower Return Rate than Version A The cart icon's color makes viewing the contents of the cart harder for the user, since they may not be able to find the cart. This could cause B to to have a lower Return Rate since users may not leave the first page in the first place.

Here is an infographic depicting my conclusions:


And as for my final takeaways:

  • 20 samples is nowhere near enough for AB testing. My p-values were way off, and a lot of it was due to the high standard deviations. Collecting more samples may help with this problem.
  • I attempted altering a simple hierarchy/consistency rule by changing the color of an icon to be the exact same as a button. With 20 samples, it doesn't seem like I can make any conclusion about this having a significant impact on navigation through a simple website. Perhaps this change applied on a larger scale could create more of a difference.
  • There's something that didn't feel right about data collection either. Since everyone was at their own home/working space during section, there was no way of controlling the conditions where the users were operating from. Perhaps a much larger sample size could alleviate this problem, but this could be one of the reasons why there was such a high standard deviation.
